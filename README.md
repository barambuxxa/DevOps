# DevOps
Данный проект представляет собой комплексное практическое руководство по внедрению современных DevOps-практик. Он охватывает полный цикл создания и управления инфраструктурой — от развертывания виртуальных машин до настройки полноценного CI/CD пайплайна с мониторингом и логированием.
Мы рассмотрим следующие темы:

- Infrastructure as Code (IaC). Использование Terraform для развёртки виртуальных машин в Proxmox
- Configuration Management с помощью Ansible для автоматической настройки серверов и развертывания Kubernetes-кластера
- Application Containerization & Packaging — создание Docker-образа и управление развертыванием через Helm.
- CI/CD Pipeline Setup — настройка автоматизированных пайплайнов с использованием Jenkins и ArgoCD (GitOps)
- Monitoring & Alerting — внедрение системы мониторинга на базе Prometheus Stack с оповещениями в Telegram

Данный проект показывает настройку с нуля, от простого к сложному:
-	Уровень 1: Физическая инфраструктура (Proxmox VE)
-	Уровень 2: Виртуальные машины (управляемые через Terraform)
-	Уровень 3: Kubernetes-кластер (развернутый с помощью Ansible)
-	Уровень 4: Приложение (упакованное в Docker и Helm)
-	Уровень 5: CI/CD пайплайны (Jenkins и ArgoCD)
-	Уровень 6: Мониторинг (Prometheus, Grafana, Alertmanager)
